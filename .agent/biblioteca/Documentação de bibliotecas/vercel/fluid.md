Efficiency gains that pay off
-----------------------------

### Server-like concurrency, in a serverless world

In-function concurrency enables a single Vercel Function to handle multiple invocations simultaneously, optimizing resource usage and turning efficiency gains into savings.

**Traditional serverless wastes idle time.**

It fails to efficiently utilize available resources during periods of inactivity.

![](https://assets.vercel.com/image/upload/v1754078989/front/fluid/fluid-serverless-light-desktop.svg)![](https://assets.vercel.com/image/upload/v1754079048/front/fluid/fluid-serverless-light-mobile.svg)![](https://assets.vercel.com/image/upload/v1754078981/front/fluid/fluid-serverless-dark-desktop.svg)![](https://assets.vercel.com/image/upload/v1754079047/front/fluid/fluid-serverless-dark-mobile.svg)

**Fluid runs invocations concurrently.**

Overall cloud resources provisioned are dramatically reduced while invocations are kept alive in-memory.

![](https://assets.vercel.com/image/upload/v1754079046/front/fluid/fluid-memory-light-desktop.svg)![](https://assets.vercel.com/image/upload/v1754079048/front/fluid/fluid-memory-light-mobile.svg)![](https://assets.vercel.com/image/upload/v1754079045/front/fluid/fluid-memory-dark-desktop.svg)![](https://assets.vercel.com/image/upload/v1754079045/front/fluid/fluid-memory-dark-mobile.svg)

**CPU that’s charged when active, not idle.**

No more paying for I/O wait times, network latency, and API calls.

![](https://assets.vercel.com/image/upload/v1754079047/front/fluid/fluid-cpu-light-desktop.svg)![](https://assets.vercel.com/image/upload/v1754079050/front/fluid/fluid-cpu-light-mobile.svg)![](https://assets.vercel.com/image/upload/v1754079046/front/fluid/fluid-cpu-dark-desktop.svg)![](https://assets.vercel.com/image/upload/v1754079049/front/fluid/fluid-cpu-dark-mobile.svg)

### In-function concurrency

Run multiple invocations on a single function instance, reducing idle compute time and lowering costs​.

![](/vc-ap-vercel-marketing/_next/static/media/function.97260ab0.svg?dpl=dpl_DAmfQ7ShxgPNUreS74eoRtAwLkga)

### Cold-start reduction

Functions are pre-warmed and optimized with bytecode caching, ensuring faster response times​.

![](/vc-ap-vercel-marketing/_next/static/media/route.e053b4ba.svg?dpl=dpl_DAmfQ7ShxgPNUreS74eoRtAwLkga)

### Streaming

Send data to users as it becomes available, improving performance for AI, media, and real-time apps​.

![](/vc-ap-vercel-marketing/_next/static/media/shield.c7dd784a.svg?dpl=dpl_DAmfQ7ShxgPNUreS74eoRtAwLkga)

### Cross-region failover

Ensures high availability by rerouting traffic to backup regions during outages​.

![](/vc-ap-vercel-marketing/_next/static/media/notebook.f186cc13.svg?dpl=dpl_DAmfQ7ShxgPNUreS74eoRtAwLkga)

### Dynamic scaling

Automatically adjusts concurrency and resource allocation based on real-time demand​.

![](/vc-ap-vercel-marketing/_next/static/media/lightning.6c0936c2.svg?dpl=dpl_DAmfQ7ShxgPNUreS74eoRtAwLkga)

### Post-response tasks

Keep functions running after sending a response to handle tasks like logging or database updates​.
